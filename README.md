# MAX Autopost (Free) v1.7.0

Fix релиз: MAX upload step2 иногда возвращает payload не как `{token,url,type}`, а как вложенный объект, например `{"photos":{...}}`.  
Плагин теперь принимает **любой валидный JSON** и передаёт его целиком в `image.payload`.

## Что нового в 1.7.0
- Добавлена отдельная настройка максимальной длины текста поста для MAX (регулируется в админке).
- Теперь можно явно ограничить длину публикации от 200 до 3900 символов.

## Что нового в 1.6.2
- По запросу удалён первый скриншот из блока подсказок на вкладке «Техпомощь».
- Оставлены только 2 скриншота: добавление бота в группу и поиск Chat ID в плагине.

## Что нового в 1.6.1
- Во вкладку «Техпомощь» добавлены встроенные скриншоты-подсказки по шагам (токен, добавление бота, поиск Chat ID).
- Скриншоты отображаются прямо в админке плагина, чтобы пользователю было проще ориентироваться.

## Что нового в 1.6.0
- Добавлена отдельная вкладка «Техпомощь» с пошаговой инструкцией по созданию бота, добавлению в группу и поиску Chat ID.
- При введенном Token вкладка пытается автоматически показать доступные Chat ID через API MAX.

## Что нового в 1.5.2
- Добавлена защита от массовой повторной отправки после обновления плагина: старые элементы очереди до момента обновления больше не улетают автоматически.
- Для повторной отправки старых записей используйте ручную постановку в очередь (кнопки/массовые действия).

## Что нового в 1.5.1
- Исправлен UX по запросу: галочки теперь для выбора **типов записей** (включая кастомные post types), а не для meta-полей.
- Блок контактов поддержки отделён от рекламного блока, чтобы не смешивать их визуально.

## Что нового в 1.5.0
- Выбор кастомных полей теперь можно делать галочками прямо в настройках (авто-подбор meta_key из контента).
- Добавлен рекламный виджет и блок контактов на странице админки: по вопросам `aleksey@krivoshein.site`.

## Что нового в 1.4.1
- Улучшена надежность авто-отправки: при постановке в очередь теперь принудительно триггерится worker (single event + loopback cron).
- Добавлен явный перехват перехода `future -> publish` для запланированных публикаций.

## Что нового в 1.4.0
- Обновлена админка очереди: быстрые действия «Отправить» и «В очередь» прямо в таблице.
- Добавлена массовая постановка в очередь всех опубликованных материалов.
- Поддержка страниц и других публичных типов записей (метабокс, queue, row/bulk actions).

## Что нового в 1.3.0
- Добавлена публикация кастомных полей в конце текста поста.
- Настройка маппинга через список `meta_key|Подпись` (или только `meta_key`).

## Features
- One message: image (optional) + text + button (optional)
- Attachments order: image first, button second
- WP-Cron queue + retry/backoff + cron lock
- Logs tab

## Install
Upload folder `max-autopost` into `/wp-content/plugins/`, activate.
