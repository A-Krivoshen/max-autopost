# MAX Autopost (Free) v1.10.2

## Что нового в 1.10.1
- Релизная проверка `message_format`: подтверждена совместимость `formatted` с multi-target отправкой, изображением и кнопкой «Читать».
- Подтверждена fallback-логика `formatted -> plain_text` на ошибках API и сохранение отправки по остальным target.
- Подготовлена release-ready упаковка (ZIP с корневой папкой `max-autopost`).

## Что нового в 1.10.0
- Добавлена настройка формата сообщения: `plain_text`, `formatted`, `excerpt_plain`.
- Для `formatted` реализована безопасная нормализация WordPress HTML: удаление Gutenberg-комментариев, whitelisting тегов и читаемая обработка списков.
- Добавлен fallback: если отправка `formatted` не проходит API, плагин автоматически повторяет отправку в `plain_text` и пишет это в лог.
- Тестовая отправка теперь учитывает выбранный формат и в режиме `formatted` отправляет демонстрационный текст (жирный/курсив/ссылка/список/code).
- Обновлена release-ready схема с сохранением совместимости GitHub updater.

## Что нового в 1.9.2
- Инициализация GitHub updater вынесена в отдельный include-класс для более чистой архитектуры.
- Добавлен `lib/plugin-update-checker/README.md` с ожидаемым путём библиотеки и поведением fallback.

## Что нового в 1.9.1
- Добавлена поддержка автообновлений плагина из GitHub Releases через `plugin-update-checker` (YahnisElsts).
- В заголовок плагина добавлен `Update URI` для корректной работы внешних обновлений вне WordPress.org.
- Добавлена безопасная инициализация update-checker: при отсутствии библиотеки плагин работает как обычно, без фатала и без лишних notice.
- Для релизов включён режим использования Release assets с фильтром ZIP (`/\.zip($|[?&#])/i`), чтобы выбирать архив плагина.

## Что нового в 1.9.0
- Добавлена multi-target отправка: теперь один пост можно отправлять сразу в несколько `chat_id` (каналы и/или группы MAX).
- Сохранена обратная совместимость: старое поле `chat_id` остаётся основным и работает как раньше.
- В настройках добавлено поле **Дополнительные Chat ID** (одно значение на строку, trim/sanitize, удаление пустых строк и дубликатов).
- Отправка выполняется последовательно по всем target: ошибка в одном чате не останавливает отправку в остальные.
- Добавлен агрегированный итог отправки: `success`, `partial_success`, `error`.
- Для каждого target сохраняется результат в метаданные поста (`chat_id`, `status`, `message_id`, `error`) и выводится в очереди.
- Кнопка «Отправить тест» теперь отправляет тест во все настроенные цели.
- Обновлены очередь и help: фильтр/счётчик `partial_success`, отображение результатов по целям и инструкция по формату списка Chat ID.

## Что нового в 1.8.9
- Добавлена поддержка `define('KRV_MAX_CHAT_ID', '...');` — теперь Chat ID, как и Token, можно вынести из базы WordPress в `wp-config.php`.
- Все отправки и тестовые сообщения используют Chat ID из `wp-config.php`, если константа задана.

Fix релиз: MAX upload step2 иногда возвращает payload не как `{token,url,type}`, а как вложенный объект, например `{"photos":{...}}`.  
Плагин теперь принимает **любой валидный JSON** и передаёт его целиком в `image.payload`.

## Что нового в 1.8.8
- На вкладке «Очередь» добавлены наглядные счетчики по статусам: `Все / queued / error / sent`.
- Счетчики кликабельны и сразу переключают фильтр, чтобы быстрее разбирать состояние очереди.

## Что нового в 1.8.7
- Во вкладке «Очередь» добавлены фильтры по статусам: `queued`, `error`, `sent` и `Все`.
- Это ускоряет разбор очереди и ошибок в админке при большом количестве публикаций.

## Что нового в 1.8.5

## Features
- One message: image (optional) + text + button (optional)
- Attachments order: image first, button second
- WP-Cron queue + retry/backoff + cron lock
- Logs tab

## Install
Upload folder `max-autopost` into `/wp-content/plugins/`, activate.
